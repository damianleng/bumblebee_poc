# BumbleBee Master Data Automation — PoC

**G7 Tech Services | March 2026 | Confidential — BumbleBee Foods**

AI Agent that automates the Partner Function Change Request workflow.
Reads incoming emails + Excel attachments, extracts structured data via Claude, presents to a human reviewer (HITL), and simulates a write-back to SAP via Skybot mock.

## Local Development Setup

### Prerequisites
- Docker + Docker Compose
- Anthropic API key
- Python 3.11+ (for generating fixtures only)

### 1. Environment

Create a `.env` file in the `poc/` root:

```env
ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=postgresql://bumblebee:bumblebee@db:5432/bumblebee_poc
```

### 2. Generate Excel fixtures (run once)

```bash
pip install pandas openpyxl
python fixtures/generate_fixtures.py
```

### 3. Start the stack

```bash
docker compose -f docker-compose.dev.yml up --build
```

Backend auto-creates tables and seeds `sap_lookup` on first startup.

### 4. Verify

```bash
curl http://localhost:8001/health
# → {"status":"ok"}
```

## AI Agent Pipeline

```
POST /api/ingest
    └── parse_excel()          # Excel → plain text table
    └── run_agent()            # Two-step Claude pipeline
        ├── Step 1: Extract    # Claude reads email + Excel → structured JSON
        └── Step 2: Validate   # Strip non-numeric account IDs, classify confidence
    └── Store to DB
        ├── requests           # One row per email
        └── request_items      # One row per extracted account change
```

**Confidence threshold:**
- `>= 0.85` → `auto_classified` (proceed to HITL review)
- `< 0.85` → `needs_review` (flag for manual triage)

---

## Database

Three tables on `rks-postgres` (or local dev postgres):

| Table | Purpose |
|---|---|
| `requests` | One row per incoming email |
| `request_items` | One row per account change within a request |
| `sap_lookup` | 13 hardcoded SAP accounts (replaces live SAP query) |

To run migration manually against an existing postgres instance:

```bash
psql -h localhost -U bumblebee -d bumblebee_poc -f migrations/001_init.sql
```

## Demo Walkthrough

### Overview

```
Ingest email + Excel → Claude extracts changes → Parsing Result screen →
Send for Review → HITL Delta screen → Approve / Deny per line →
Submit to SAP (Skybot mock) → status = completed → Audit Log
```

**Screens involved:**
- `/` — Dashboard: all requests, status badges, Review button per row
- `/requests/:id` — Parsing Result: AI extraction card + extracted table
- `/requests/:id/review` — HITL Delta: SAP current vs proposed, per-item approve/deny
- `/audit` — Audit Log: full record with timestamps, filterable by status

### Fixture 3 — Ambiguous Email (No Excel)

**Scenario:** Procurement sent a vague email — no account IDs, no attachment, unsure which field.

**Ingest:**
```bash
curl -X POST http://localhost:8001/api/ingest \
  -F "sender=procurement.user@bumblebeefoods.com" \
  -F "subject=Account changes needed" \
  -F "email_body=$(cat fixtures/fixture_3_ambiguous/email.txt)"
```

**What Claude extracts:**
- Confidence: **~30%** → `needs_review`
- Items: **0** — no valid account IDs found
- Notes: explains uncertainty (field unclear, no account numbers provided)

**Parsing Result screen:**
- Confidence badge shows **30% orange** → `needs_review`
- Extracted table is empty
- Two paths:
  - **"Send for Review"** → HITL screen with empty table, human handles manually
  - **"Flag for Manual Review"** → status = `flagged`, back to Dashboard

**Purpose in the demo:** Shows the AI does not hallucinate account IDs. When it doesn't know, it says so — and routes to a human.
