# BumbleBee Master Data Automation ‚Äî PoC

**G7 Tech Services | March 2026 | Confidential ‚Äî BumbleBee Foods**

AI Agent that automates the Partner Function Change Request workflow.
Reads incoming emails + Excel attachments, extracts structured data via Claude, presents to a human reviewer (HITL), and simulates a write-back to SAP via Skybot mock.

## Local Development Setup

### Prerequisites
- Docker + Docker Compose
- Anthropic API key
- Python 3.11+ (for generating fixtures only)

### 1. Environment

Create a `.env` file in the `poc/` root:

```env
ANTHROPIC_API_KEY=sk-ant-...
DATABASE_URL=postgresql://bumblebee:bumblebee@db:5432/bumblebee_poc
```

### 2. Generate Excel fixtures (run once)

```bash
pip install pandas openpyxl
python fixtures/generate_fixtures.py
```

### 3. Start the stack

```bash
docker compose -f docker-compose.dev.yml up --build
```

Backend auto-creates tables and seeds `sap_lookup` on first startup.

### 4. Verify

```bash
curl http://localhost:8001/health
# ‚Üí {"status":"ok"}
```

## AI Agent Pipeline

```
POST /api/ingest
    ‚îî‚îÄ‚îÄ parse_excel()          # Excel ‚Üí plain text table
    ‚îî‚îÄ‚îÄ run_agent()            # Two-step Claude pipeline
        ‚îú‚îÄ‚îÄ Step 1: Extract    # Claude reads email + Excel ‚Üí structured JSON
        ‚îî‚îÄ‚îÄ Step 2: Validate   # Strip non-numeric account IDs, classify confidence
    ‚îî‚îÄ‚îÄ Store to DB
        ‚îú‚îÄ‚îÄ requests           # One row per email
        ‚îî‚îÄ‚îÄ request_items      # One row per extracted account change
```

**Confidence threshold:**
- `>= 0.85` ‚Üí `auto_classified` (proceed to HITL review)
- `< 0.85` ‚Üí `needs_review` (flag for manual triage)

## Database

Three tables on `rks-postgres` (or local dev postgres):

| Table | Purpose |
|---|---|
| `requests` | One row per incoming email |
| `request_items` | One row per account change within a request |
| `sap_lookup` | 13 hardcoded SAP accounts (replaces live SAP query) |

To run migration manually against an existing postgres instance:

```bash
psql -h localhost -U bumblebee -d bumblebee_poc -f migrations/001_init.sql
```
